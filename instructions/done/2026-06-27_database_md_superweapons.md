# 指示書: DATABASE.md 超兵器セクション作成

- 発行者: Claude Code
- 発行日: 2026-06-27
- 担当: Codex
- ステータス: 未対応

---

## 目的

ACM-MDプロジェクト(`MOD_OVERVIEW.md`参照)の最初の典拠台帳として、
`DATABASE.md` を新規作成し、種別①「超兵器」セクションを完成させる。
本DBはMOD実装の唯一の典拠台帳であり、ここに無い情報をMOD実装に使うことは禁止。

---

## 作業内容

1. **`INSTRUCTION_FOR_DATABASE.md` を必ず全文読み込むこと**
   本指示書はその要約ではなく着手指示。詳細な記載ルール・スキーマ・
   典拠優先順位・ハルシネーション防止ルールは全て元指示書に従う。
2. リポジトリルートに `DATABASE.md` を新規作成する
   (まだ存在しない場合。存在する場合は末尾に追記)
3. 対象11作品(AC2 → AC3 → AC04 → AC5 → ACZ → ACX → AC6 → ACJA → AH → AC7 → ACI の発表順)から、
   公式(acecombat.jp の ACES CHRONICLE)が認定する超兵器
   (CBRN含む superweapon リスト掲載のもの)を洗い出す
4. 各エントリは `INSTRUCTION_FOR_DATABASE.md` 4章の項目スキーマに厳密に従って記述する
   - 名称、種別、登場作品、製造主体、性質、典拠URL、HOI4実装判定、判定根拠、
     MD既存実装との対応、備考の全フィールド必須
5. MD既存実装との対応は `common/` 配下を実際に `grep` して確認し、
   ファイルパス+行番号、または「該当なし(grep済み: キーワード)」を記録する
   (このリポジトリにMD本体のコードが無い場合は「未確認(MD実コード未取得)」と明記してよい)
6. 固定設置型超兵器(ストーンヘンジ、エクスキャリバー、メガリス等)は
   判定`× 除外`だが、エントリ自体は作成する(判断材料として残すため)

---

## 典拠ルール(厳守・要約)

詳細は `INSTRUCTION_FOR_DATABASE.md` 3章・7章を参照。特に重要な点:

- **超兵器分類は公式(acecombat.jp の ACES CHRONICLE)を最優先**。
  acecombat.wiki.gg の Disputed セクションの独自判定は採用しない
- 典拠URLが提示できないエントリは作らない(例外なし)
- 「~らしい」「~と思われる」等の推測表現は禁止。不明点は「未確認」と明記
- AIの「記憶」に頼った記述を禁止。確実にURLで確認できた情報のみ書く

## 判断に迷ったら止まって質問すべきケース

- 設定上「量産されなかった1点物」かつ「機体本体」のもの(△か×かでブレる)
- MD既存実装と機能が被るもの
- 典拠URLは見つかったが内容が公式と二次創作で食い違うもの
- 公式とacecombat.wiki.ggで超兵器分類が割れている場合

該当する場合は自己判断で進めず、TODO.md の「ブロック中のタスク」に追記して
作業を止め、ユーザの判断を仰ぐこと。
