## 作業内容

### 1. 調査対象パス

Steam β版 `3374271790` の以下:

- `gfx/interface/` 配下(モジュール・装備アイコンの .dds 実体)
- `interface/` 配下(.gfx 定義ファイル)
- 比較対象として、モジュール定義側の参照(`abbreviation` / `picture` 等で参照
  される sprite 名)

### 2. 抽出すべき情報

以下を `KNOWLEDGE.md`「MDモジュール定義書式」セクション直下に **新サブセクション
「アイコン .dds 仕様 + .gfx 登録」** として追加する:

#### (a) アイコン .dds の物理仕様
- **サイズ**(モジュールアイコン用の標準ピクセルサイズ。例: 64x64, 128x128 等)
- **圧縮形式**(DXT1 / DXT3 / DXT5 / 非圧縮など。実例の .dds ヘッダから判定)
- **カラーチャンネル**(RGB / RGBA / グレースケール)
- **mipmap 有無**(MDの慣例)
- 実際にどのサイズ・圧縮が使われているかを **既存 .dds ファイル数本のサンプル**
  から抽出して記録(file コマンド or バイナリヘッダ確認)

#### (b) .gfx ファイルでの sprite 登録書式
- 基本書式:
  ```
  spriteType = {
    name = "GFX_<sprite_name>"
    texturefile = "gfx/interface/<...>.dds"
  }
  ```
  あるいはMD独自パターンがあれば実例で
- sprite 名の **命名規則**(例: `GFX_<module_id>` の規則性、prefix の有無)
- どの .gfx ファイルに何が登録されているか(モジュール系 / アーキタイプ系 /
  技術ツリー系 で分かれているか)

#### (c) モジュール定義側からの参照
- モジュール定義の中で sprite が指定される箇所(例: `picture = <sprite_name>`)
  と、その有無
- 「指定なしの場合、自動命名(`GFX_<module_id>` 等)で解決されるか」を
  実例で確認

#### (d) アーキタイプ・variant のアイコン
- アーキタイプ側で `sprite = <名前>` のように指定されている例
  (mothership_equipment は `sprite = mothership` を持っていた、と
  Claude Code が既に確認済み — その全体パターンを抽出)
- variant 側の sprite 指定有無

#### (e) Special Project 関連のアイコン(任意、余裕があれば)
- Special Project にアイコンがある場合の指定方法
  (P3-C で本格扱いするが、命名・参照だけここで触れて良い)

### 3. 加えて記録すべき注意点

- 「.gfx 1ファイル sprite 数 約300未満」のラインを本MOD実装で守るための
  「本MOD用 .gfx ファイル分割案」を SPEC.md 側に1-2行で書く
- 「.dds は BC1 (DXT1) / BC3 (DXT5) どちらが MD で標準か」を抽出する

  **モジュール定義側でも同じか** を実例で確認(モジュールで `picture` が
  有効か無効か)

## 判断に迷ったら止まって質問すべきケース

- .dds のサイズ・圧縮形式が MD 内で統一されておらず、複数規格が混在している場合
  (どれを「本MOD標準」とすべきかは本人判断)
- sprite 命名規則が予想と大きく食い違う場合
- ImageMagick / DirectXTex 等の追加ツールが必要そうな場合
  (Codex 環境で .dds の中身を確認する手段がない場合は、`file` コマンドや
  バイナリ先頭の DDS ヘッダだけでも記録すれば十分。ピクセルサイズの確認
  だけなら標準ツールでも可能)
