# PLAN.md — やりたいことの整理

本ドキュメントは、ACモジュールMOD (ACM-MD) の動機・狙い・方向性を
テキストで整理したものである。
SPEC.md(仕様)に進む前段階の、思考の素材としての位置づけ。

---

## なぜこのMODを作りたいか

Millennium Dawn を陸軍主体で大国(米中露+日本)プレイしていて、
装甲・機械化師団で物量で押すスタイルが楽しい。
ただ、戦争そのものに集中していると、航空・ミサイルとの連携が
物足りなく感じる場面が増えてきた。

特に近接航空支援(CAS)を多めに運用したいが、
バニラ/MDのCASは「制空を取った州で陸戦に修正値が乗る」程度の
抽象表現で、見ていて満足感が薄い。

加えて、自分は「好きな装備を組んで、大量配備して敵を押し潰す」プレイが
何より楽しいと感じている。装備の性能数値・名前・組み合わせの妙に
時間を使うのが好き。

つまり「自分のプレイスタイルに合う道具を増やす」こと自体が
本MODの目的になる。

---

## 何を作るか(粗い輪郭)

エースコンバットシリーズに登場する超兵器・架空兵器の特徴的システムを
モジュールとして MD に追加し、プレイヤーが既存のMD装備に
「ACらしさ」を載せて運用できるようにする。

具体例(あくまで方向性、確定ではない):
- アリコーンの電磁レールガン砲塔を艦船モジュールとして追加 →
  既存の戦艦・巡洋艦に載せて「アリコーン風」の主砲構成を作れる
- アーセナルバードの随伴ドローン関連モジュールを追加 →
  MDに既に存在する `mothership_equipment` の派生として運用
- 各作品の特徴的システム(COFFIN、APS、TLS等)をモジュール化

機体本体・艦本体・車両本体は新規追加しない。
アイコンを大量に作れないという制約があるため、
モジュール側に集中する。

---

## 方針として決まっていること

- **メインはモジュール追加**(既存アーキタイプのスロットを拡張する)
- **機体・艦・車両の本体は新規追加せず**、既存品にモジュールで個性を載せる
- **例外**: 1点物の超兵器のみ、Special Projects 経由の専用アーキタイプを
  追加可能(MDが既に採用しているパターン)
- **入手条件**: 研究で解放
- **対象作品**: ACシリーズ全体を視野に入れるが、深掘りは1作品に絞る
  (どの作品を本命にするかは未定)
- **性能チューニング**: 原作寄りだが、コスト・製造難易度・必要技術で
  ゲームバランスを取る
- **陣営性(オーシア/エルジア等の固有化)**: 初期スコープでは無視
  (全文明で同じモジュール群を解放可能)

---

## やらないと決めたこと

- AC作品の通常兵装(4AAM、6AAM、QAAM、SAAM、マルチロックオン系)の追加
- 固定設置型超兵器(ストーンヘンジ、エクスキャリバー等)のモジュール化
- 量産戦闘機の本体追加(MDの既存実装で十分)
- 架空車両・実在車両・実在艦のDB化(本MODの対象外)
- 機体・艦・車両本体のアイコン新規作成
- 上陸作戦・揚陸艦の強化(別案として検討したが本人の関心薄)

---

## まだ決まっていないこと(SPEC策定までに固める)

- **本命作品の選定**: 対象11作品の中からどれを最初の深掘り対象にするか
- **モジュールカテゴリの粒度**: 「超兵器砲モジュール」のような一括カテゴリにするか、
  各超兵器ごとに別カテゴリにするか
  (現時点の合意: 別カテゴリ)
- **誰が積めるか**: 全戦闘機/艦に許可するか、ステルス機限定など条件を付けるか
  (MDの仕様確認後に決定)
- **AC既存超兵器とMD既存兵器の棲み分けルール**: 数値帯の決め方
- **モジュール用アイコンの調達方法**: 既存流用で本当に満足できるか、
  将来的にAI生成等を検討するか
- **MOD名(正式)**: リポジトリ名 `ace-combat-modules-md` をそのまま使うか、
  ゲーム内表示用に別途付けるか

## 開発の進め方

二段構えのワークフロー:

1. **戦略・調査・指示書作成**: claude.ai(ブラウザ/モバイル)
   - 思考の整理、PLAN/SPEC/TODO の更新
   - AC設定の典拠調査、指示書作成
   - MDコードの軽い確認(GitHub経由)
2. **実コード調査・実装**: Claude Code または Codex(PC)
   - DATABASE.md の埋め(典拠ベース)
   - MD実コードの調査(grep、ファイル読み込み)
   - モジュール定義の実装、アイコン配置、Localization
   - ナレッジは KNOWLEDGE.md に蓄積

MD はバニラから大幅独自実装のため、Claude Code / Codex には
「推測実装させず、まず実コードを読ませて事実確定 → 実装」
の二段構えを徹底する。

Claude Code と Codex は同じプロジェクトルールに従い、
依頼により使い分ける。両者の同時並行作業は禁止。

## このMODが成功した状態のイメージ

- 「F-22A の機体に COFFIN モジュール + マルチロック兵装相当の特殊兵装 を
  載せて "ADF-01 FALKEN 風" を作る」のような遊びがゲーム内で成立する
- アーセナルバード相当の機体を、MD既存実装とAC由来の追加モジュールで
  「自分なりの構成」で組み立てられる
- 通常プレイの装備群と混在しても破綻せず、研究・コスト・製造難度で
  自然な選択肢になっている
- 追加モジュールの根拠(どのAC作品のどの兵器か)が DATABASE.md に
  記録され、再現性とハルシネーション防止が両立している
